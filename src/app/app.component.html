Investigación: Software y funcionamiento conversacional en el robot NAO

1. Introducción

El robot NAO, desarrollado por SoftBank Robotics, es un humanoide programable orientado a investigación, educación y experimentación en interacción humano-máquina. Su arquitectura combina un sistema operativo embebido, un framework de control y herramientas de desarrollo que permiten construir comportamientos, diálogos y acciones mediante lógica visual y código.

2. Software utilizado en NAO

El ecosistema de software de NAO está compuesto por las siguientes herramientas principales:

Software / Herramienta Función principal
NAOqi OS Sistema operativo embebido basado en Linux que ejecuta los servicios centrales del robot.
NAOqi Framework Framework que expone APIs para sensores, actuadores, visión, audio, movimiento y demás funcionalidades.
Choregraphe Entorno visual (IDE) para diseñar comportamientos mediante diagramas de flujo (bloques), animaciones y cajas personalizadas.
SDK (Python y C++) Paquetes para programar el robot mediante código, permitiendo integraciones y lógicas complejas.
ALDialog Módulo de NAOqi para crear diálogos basados en reglas y reconocimiento de expresiones clave.
ALSpeechRecognition / ALSpeechSynthesis Servicios para reconocimiento de voz y síntesis de habla (TTS) respectivamente.

Nota: NAO emplea mayoritariamente sistemas basados en reglas y plantillas para diálogo; no integra nativamente modelos generativos tipo LLM.

3. Funcionamiento del sistema de conversación

El sistema conversacional de NAO ejecuta las siguientes etapas:

  1. Captura de voz mediante los micrófonos integrados.
  2. Procesamiento mediante reconocimiento de palabras clave o frases previamente definidas.
  3. Comparación con reglas almacenadas en archivos de diálogo (.top, .dlg) o en ALDialog.
  4. Ejecución de la respuesta a través de síntesis de voz y/o acciones físicas (animaciones, movimientos).

Ejemplo de regla conversacional (sintaxis típica):

u:(hola) ¡Hola humano! ¿Cómo estás?
u:(como te llamas) Me llamo NAO, tu robot favorito.

Si la entrada del usuario no coincide con ninguna regla conocida, la respuesta será por defecto limitada o de tipo "no entiendo".

4. Diagramas de flujo en Choregraphe

Choregraphe permite construir la lógica de interacción mediante nodos y cajas conectadas. Cada caja representa una acción, un bloque de reconocimiento, o una decisión lógica. Esto facilita el diseño de comportamientos sin necesidad de escribir código.

Tipos principales de bloques

  • Say — El robot emite una frase.
  • Animated Say — Habla acompañado de movimientos predefinidos.
  • Speech Recognition — Detecta palabras clave o frases.
  • Switch / If — Decisiones condicionales.
  • Animation — Ejecuta una animación corporal.
  • Custom Box — Bloque personalizado con lógica propia (puede contener Python).

7. Conclusiones

  • NAO opera sobre un sistema basado en Linux (NAOqi OS) y un framework que orquesta sensores y actuadores.
  • Su sistema conversacional se apoya en reglas predefinidas y reconocimiento de patrones; no incorpora LLMs de forma nativa.
  • Los diagramas de flujo en Choregraphe facilitan el diseño modular y visual de interacciones complejas.
  • Es posible simular y controlar comportamientos desde un computador externo (por ejemplo, una Raspberry Pi) usando el SDK y las APIs de NAOqi.
